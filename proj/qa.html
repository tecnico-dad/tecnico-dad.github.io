Dúvidas 1ª Parte do Projecto

  1. Pergunta: O Binas deve registar-se no UDDI?

    Resposta: Sim e deve chamar-se tal como é referido no enunciado.


  2. Pergunta: É necessário fazer testes para as operações auxiliares ?

    Resposta: No enunciado refere-se que as operações auxiliares - ping, clear, ... - se "destinam a auxiliar os testes e não necessitam de ser testadas exaustivamente".
    Sendo mais concreto, podem fazer testes para estas operações, mas estes testes não serão avaliados.

    Já as operações principais podem e devem ser testadas o melhor possível.


  3. Pergunta: Como se executam os testes de integração?

    Resposta: Para executar os testes de integração:

    • Lançar o servidor:
      cd station-ws
      mvn compile exec:java
    • Correr os testes de integração:
      cd station-ws-cli
      mvn verify


  4. Pergunta: Para que serve o station-ws-cli?

    Resposta: O projeto station-ws-cli serve para:

    • Gerar os stubs do station-ws
    • Conter os testes de integração (...IT) do station-ws
    • Encapsular criação de stubs e invocações remotas a um servidor station-ws dentro da classe StationClient.


  5. Pergunta: O que deve fazer a classe ...ClientApp?

    Resposta: Esta classe deve ter um método static main que serve apenas para fazer uma invocação simples do servidor. Sugere-se que faça um ping() e que imprima os resultados para a consola.
    Para executar, fazer: mvn compile exec:java e ver o resultado do ping.
